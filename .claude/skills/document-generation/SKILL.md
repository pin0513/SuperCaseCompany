# Document Generation

## Purpose

Generate professional, well-formatted documents in PDF, DOCX, and PPTX formats for client deliverables, proposals, presentations, and reports. Ensure consistent branding, quality, and accessibility.

## When to Use This Skill

Use this skill when:
- Creating client proposals and presentations
- Generating requirements specifications
- Producing design documentation
- Creating technical architecture documents
- Preparing project status reports
- Developing training materials
- Compiling final project deliverables

## Document Types and Formats

### PDF Documents
**Use for**: Final deliverables, formal reports, contracts
**Characteristics**: Non-editable, preserved formatting, universal compatibility
**Tools**: Export from Word/PowerPoint, Adobe Acrobat, Markdown to PDF

### DOCX Documents
**Use for**: Specifications, requirements, collaborative editing
**Characteristics**: Editable, supports comments and track changes, structured content
**Tools**: Microsoft Word, Google Docs (export), LibreOffice Writer

### PPTX Presentations
**Use for**: Client presentations, design proposals, project updates
**Characteristics**: Visual storytelling, slide-based, supports multimedia
**Tools**: Microsoft PowerPoint, Google Slides (export), Keynote (export)

## Document Quality Standards

### Content Quality
- Clear and concise writing
- Proper grammar and spelling
- Consistent terminology
- Logical structure and flow
- Appropriate level of detail

### Visual Quality
- Professional formatting
- Consistent fonts and colors
- Proper use of headings and hierarchy
- Effective use of visuals (diagrams, charts, screenshots)
- Adequate white space

### Branding
- Company logo and branding elements
- Consistent color scheme
- Standard header and footer
- Copyright and confidentiality notices
- Version and date information

### Accessibility
- Proper document structure (headings, lists)
- Alt text for images
- Readable font sizes and contrast
- Logical reading order
- Accessible PDF features

## Document Generation Process

### Step 1: Define Purpose and Audience
- What is the document's purpose?
- Who will read it?
- What format is most appropriate?
- What level of detail is needed?

### Step 2: Gather Content
- Collect all relevant information
- Review requirements, designs, specifications
- Gather supporting materials (screenshots, diagrams)
- Identify stakeholder contributions

### Step 3: Create Outline
- Structure content logically
- Define sections and subsections
- Allocate content to sections
- Identify visuals and diagrams needed

### Step 4: Draft Content
- Write content for each section
- Create or insert diagrams and visuals
- Add tables, charts, and code samples
- Maintain consistent style and tone

### Step 5: Apply Formatting
- Apply document template and branding
- Format headings, lists, and tables
- Position images and diagrams
- Add header, footer, page numbers
- Generate table of contents

### Step 6: Review and Refine
- Proofread for errors
- Verify all sections are complete
- Check formatting consistency
- Validate links and cross-references
- Ensure visuals are clear and labeled

### Step 7: Export and Package
- Export to target format (PDF, DOCX, PPTX)
- Verify exported document quality
- Check file size and compatibility
- Package with supporting files if needed
- Upload to deliverable repository

## Best Practices

### For DOCX Documents
- Use styles for headings (Heading 1, Heading 2, etc.)
- Enable track changes for collaborative editing
- Use comments for review feedback
- Insert table of contents with automatic updates
- Use cross-references for section links

### For PDF Documents
- Export from source with high quality
- Enable bookmarks from headings
- Embed fonts for consistency
- Add metadata (title, author, keywords)
- Optimize file size for sharing

### For PPTX Presentations
- One key message per slide
- Use visuals over text
- Consistent slide layouts
- Speaker notes for context
- Export to PDF for distribution

## Collaboration Features

### Collaborative Editing
- Use cloud-based tools (Google Docs, Office 365)

### Review and Approval

## Success Criteria

Document generation succeeds when:
- Documents are professionally formatted and branded
- Content is clear, accurate, and complete
- Format is appropriate for purpose and audience
- Documents are accessible and well-structured
- Review and approval process is efficient
- Final documents meet quality standards
- Deliverables are packaged and delivered on time

## Example Usage

**Scenario**: Generate requirements specification for e-commerce project

```
Step 1: Define
- Purpose: Document functional and non-functional requirements
- Audience: Design team, development team, client stakeholders
- Format: DOCX for collaboration, PDF for client delivery

Step 2: Gather
- Client meeting minutes (2026-02-11, 2026-02-15, 2026-02-20)
- Market research report
- Competitive analysis
- User personas
- Traceability matrix

Step 3: Outline
1. Executive Summary
2. Project Overview
3. Functional Requirements (user management, product catalog, shopping cart, checkout, payment)
4. Non-Functional Requirements (performance, security, mobile)
5. Acceptance Criteria
6. Appendices (traceability matrix, glossary)

Step 4: Draft
- Write each section based on gathered materials
- Include user stories and use cases
- Add acceptance criteria for each requirement
- Insert diagrams for workflows

Step 5: Format
- Apply company requirements template
- Format with proper headings (Heading 1, Heading 2)
- Insert table of contents
- Add header/footer with project name, version, date
- Insert company logo

Step 6: Review
- Requirements Director review
- Requirements Consultant review
- Incorporate feedback
- Version: v1.1.0-review

Step 7: Export
- Export to PDF for client presentation
- Keep DOCX version for edits
- Upload both to deliverables/client-name/project/requirements/
- Version: v2.0.0-client
```
